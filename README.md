# Developer Home Assessment

Thank you for your interest in this opportunity.

Please review the tasks below and complete the one(s) that best match your technical expertise. The purpose of this assessment is to evaluate your practical development skills, code quality, and ability to work with an existing codebase.

## Submission Requirements

* Complete the selected task(s).
* Record a short demonstration video (3–5 minutes).
* Share your source code and video upon completion.
* Frontend implementation is **not required** unless you choose to add one.

# Task 1: Smart Contract API Integration

**Estimated Time:** 40 Minutes

### Objective

Create a new API endpoint within the shared backend project that retrieves data from an existing smart contract.

### Requirements

1. Create a new API named:

```text
[YourName]ApiTest
```

2. Connect to any deployed smart contract.
3. Fetch contract data and return it through the API.
4. Display the result via API response or console output.

### Deliverables

* Source code
* Short demo video showing:

  * API endpoint
  * Smart contract connection
  * Returned data

# Task 2: Smart Contract Development & Integration

**Estimated Time:** 40 Minutes

### Objective

Develop a smart contract and integrate it with the backend project.

### Requirements

1. Create a smart contract named:

```text
[YourName]Contract.sol
```

2. The contract must include:

   * One read function
   * One write function

3. Deploy the contract to a public test network (e.g., Sepolia or BSC Testnet).

4. Integrate the contract with the backend project.

5. Create API endpoint(s) to interact with the contract.

### Deliverables

* Smart contract source code
* Deployment details
* Backend integration code
* Short demo video showing:

  * Contract deployment
  * API interaction
  * Successful read/write operations

# Task 3: Notes REST API

**Estimated Time:** 30 Minutes

### Objective

Create a simple CRUD API for a Notes application.

### Required Endpoints

```http
POST   /notes
GET    /notes
GET    /notes/:id
PUT    /notes/:id
DELETE /notes/:id
```

### Requirements

* Use an in-memory data structure (array/object).
* No database is required.
* Demonstrate all CRUD operations through API responses or console output.

### Deliverables

* Source code
* Short demo video showing all endpoints working correctly
